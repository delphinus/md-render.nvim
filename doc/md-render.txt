INTRODUCTION						*md-render-introduction*

md-render.nvim is a Markdown rendering engine for Neovim.  It transforms raw
Markdown into richly highlighted, interactive content right inside your
editor.  Supports floating windows, tab views, in-place toggle between
source and render mode (|:MdRender-toggle|), and a pager mode for `less`-like
usage from the command line.

Features: ~

- Rich inline formatting (bold, strikethrough, inline code, links,
  Obsidian`==highlight==`)
- Tables with box-drawing borders, column alignment, and proportional sizing
- Callouts and folds (GitHub / Obsidian alert types with colored borders,
  icons, and click-to-toggle folding)
- Code blocks with treesitter syntax highlighting; expandable when truncated
- Images (PNG, JPEG, WebP, GIF, animated GIF) displayed inline via terminal
  graphics protocol
- Video (MP4, WebM, MOV, AVI, MKV, M4V) played as animated frames inline
- Mermaid diagrams rendered as images inline
- CJK-aware word wrapping (JIS X 4051 kinsoku shori + optional BudouX phrase
  segmentation)
- Clickable links (mouse click to open URLs; OSC 8 hyperlink support)
- `<details>` support (collapsible sections with click-to-toggle)
- Library API for programmatic use from other plugins

==============================================================================
REQUIREMENTS						*md-render-requirements*

- Neovim >= 0.12 (uses |nvim_ui_send()| for terminal writes)
- A terminal that supports the Kitty graphics protocol (required for
  inline images and video):
  - WezTerm (https://wezfurlong.org/wezterm/)
  - Kitty (https://sw.kovidgoyal.net/kitty/)
  - Ghostty (https://ghostty.org/)

Using lazy.nvim: >lua

  {
    "delphinus/md-render.nvim",
    version = "*",
    dependencies = {
      { "nvim-tree/nvim-web-devicons", version = "*" },
      { "delphinus/budoux.lua", version = "*" },
    },
    keys = {
      {
        "<leader>mp",
        "<Plug>(md-render-preview)",
        desc = "Markdown preview (toggle)",
      },
      {
        "<leader>mt",
        "<Plug>(md-render-preview-tab)",
        desc = "Markdown preview in tab (toggle)",
      },
      {
        "<leader>md",
        "<Plug>(md-render-demo)",
        desc = "Markdown render demo",
      },
    },
  }
<

MAPPINGS						    *md-render-mappings*

This plugin provides `<Plug>` mappings but does NOT set any default keybindings.
Map them yourself: >lua

  vim.keymap.set("n", "<leader>mp", "<Plug>(md-render-preview)")
  vim.keymap.set("n", "<leader>mt", "<Plug>(md-render-preview-tab)")
  vim.keymap.set("n", "<leader>md", "<Plug>(md-render-demo)")
<

						     *<Plug>(md-render-preview)*
<Plug>(md-render-preview) ~
	Toggle a floating preview window for the current Markdown buffer.  If
	the preview window is already open, it will be closed.

						 *<Plug>(md-render-preview-tab)*
<Plug>(md-render-preview-tab) ~
	Toggle a tab preview for the current Markdown buffer.  If the tab
	preview is already open, it will be closed.

						     *<Plug>(md-render-toggle)*
<Plug>(md-render-toggle) ~
	Toggle the current window between source and render mode for the
	current Markdown buffer.  No new window or tab is opened — the
	window's buffer is swapped in place.  See |:MdRender-toggle|.

							       *<Plug>(md-render-auto)*
<Plug>(md-render-auto) ~
	[EXPERIMENTAL] Toggle Insert-aware auto mode for the current buffer:
	render outside Insert mode, source while editing.  See
	|:MdRender-auto|.

							    *<Plug>(md-render-split)*
<Plug>(md-render-split) ~
	Open a split window showing the source and rendered views of the
	current Markdown buffer.  See |:MdRender-split|.

							*<Plug>(md-render-demo)*
<Plug>(md-render-demo) ~
	Show a demo window with all supported Markdown notations.

Inside the preview window, the following keys are available:

	`q` / `<Esc>`	Close the preview window
	`<CR>`		Toggle callout fold / expand truncated region
	`<LeftMouse>`	Click links, toggle folds, expand regions

Inside a render-mode buffer opened with |:MdRender-toggle|, `q` / `<Esc>` /
`<CR>` are NOT bound to close — call |:MdRender-toggle| again to return to
source mode.  `<LeftMouse>` still toggles folds, expands regions, and
opens links.

COMMANDS						    *md-render-commands*

The plugin exposes a single user command, |:MdRender|, with subcommands.
The legacy top-level commands (`:MdRenderTab`, `:MdRenderToggle`, etc.) are
still registered as deprecated aliases — see |md-render-deprecated-commands|.

								     *:MdRender*
:MdRender [{subcommand}] [{args}] ~

	Render the current Markdown buffer.  The {subcommand} selects the
	display mode:

	  float       - floating preview window (default; same as bare
	                |:MdRender|)
	  tab         - toggle preview in a new tab
	  pager       - pager mode (full-screen, `q` to quit Neovim)
	  toggle      - in-place source ↔ render swap in the current window
	  split       - split source + render side-by-side; honours
	                `:vert`, `:tab`, `:topleft`, `:botright`
	  auto        - [EXPERIMENTAL] auto-toggle around Insert mode;
	                takes a second arg `on` / `off` / `toggle`
	                (defaults to `toggle`)
	  demo        - show a demo window with every supported notation

	Tab completion lists the subcommands for the first arg, and
	`on` / `off` / `toggle` after `auto`.

	The per-subcommand sections below describe behaviour in detail.

							     *:MdRender-float*
:MdRender float ~
	Toggle a floating preview window for the current Markdown buffer.
	This is the default if no subcommand is given (bare |:MdRender|).

							       *:MdRender-tab*
:MdRender tab ~
	Toggle a tab preview for the current Markdown buffer.

							    *:MdRender-toggle*
:MdRender toggle ~
	Toggle the current window between source and render mode in place
	(no new window or tab).  Useful when working with splits — for
	example, edit code in one split while showing the rendered README in
	the other.

	Behavior:

	- The render buffer is read-only and reused across toggles (cached
	  per source buffer; one render buffer per source).
	- When the source buffer is shown in multiple windows, only the
	  invoking window swaps; edits from any other window are reflected
	  in the render buffer on the next toggle to render mode.
	- Cursor position round-trips: the source line nearest the cursor in
	  render mode becomes the cursor target when toggling back, and vice
	  versa.
	- 'number', 'relativenumber', and 'list' are turned off on
	  render-mode windows.  The originals are stashed on the window and
	  restored when toggling back to source.

							      *:MdRender-auto*
:MdRender auto [on|off|toggle] ~
	[EXPERIMENTAL] This feature is new and the UX may change.  Please
	report any issues or rough edges.

	Auto-toggle the current buffer between source and render based on
	Insert mode.  The state is per buffer and recorded in
	`b:md_render_auto`.

	- `:MdRender auto on` flips the current buffer to render mode
	  immediately, and registers `InsertEnter` / `InsertLeave` autocmds
	  that swap to source on entry and back to render on leave.
	- `:MdRender auto off` removes the autocmds and the buffer marker.
	  If the current window is showing the render view of this buffer,
	  it also swaps back to source so the buffer ends up in the same
	  state it had before `:MdRender auto on`.
	- `:MdRender auto toggle`, or `:MdRender auto` with no second arg,
	  toggles between on and off.  Calling it twice returns the buffer
	  to its original (auto-off, source displayed) state.

	The plugin does not register a |FileType| autocmd of its own.  To
	enable auto mode for every Markdown buffer, add one yourself: >vim

	  autocmd FileType markdown silent! MdRender auto on
<
	Behavior:

	- The render buffer is `nomodifiable`, so |InsertEnter| cannot fire
	  on it.  While auto mode is active, the keys `i` / `I` / `a` / `A`
	  / `o` / `O` are remapped on the render buffer to swap to source
	  first and then enter Insert at the corresponding position.
	- `<Esc>` returns to source's Normal mode; the |InsertLeave|
	  autocmd then debounces a swap back to render after 50 ms.
	- Rapid `i<Esc>i<Esc>` keystrokes coalesce into the final state.
	- Only the current window is swapped.  Other windows showing the
	  same source need a manual |:MdRender-toggle|.

	What works on the render buffer: ~

	  Read-only operations: motion (hjkl, gg, G, ...), search (/, ?),
	  yank (y), visual selection, marks, and <LeftMouse> (folds,
	  expand, link open).

	  `:w` and `:w!` are forwarded to the source buffer so you can
	  save without leaving render mode.  Different filenames (e.g.
	  `:w other.md`, `:saveas`) are not supported — switch to source
	  with |:MdRender-toggle| first.

	  The render buffer is named `<source-path> [render]` so it's
	  identifiable in your statusline and pickers.

	What does NOT work on the render buffer: ~

	  The render buffer is `nomodifiable`, so any mutating operation is
	  blocked or silently a no-op:

	  - Editing keys: c C s S r R d D x X p P J ~ >> <<
	  - Undo / redo (u / <C-r>) — the render buffer has no edit
	    history of its own
	  - Ex commands that mutate the buffer: :s, :!filter, gq
	  - Macros that contain editing keys

	  Run |:MdRender-toggle| (or, in auto mode, simply press `i` / `I`
	  / `a` / `A` / `o` / `O`) to swap to source first.

							     *:MdRender-split*
:MdRender split ~
	Open a split window so the source Markdown buffer and the rendered
	view are visible at the same time.  Direction follows the standard
	Vim split modifiers: >vim

	  :MdRender split          " horizontal split
	  :vert MdRender split     " vertical split
	  :tab MdRender split      " split inside a new tab
	  :topleft MdRender split  " place at the top
	  :botright MdRender split " place at the bottom
<
	Behavior is symmetric:

	- From a source window, the new split shows the render.
	- From a render window (created by |:MdRender-toggle|), the new
	  split shows the source.

	Either entry point yields a source/render pair in adjacent
	windows.

	The render buffer is shared with the |:MdRender-toggle| session
	cache, so a subsequent |:MdRender-toggle| reuses the same render
	buffer and edits propagate via the same 150 ms debounced
	live-update path used by toggle.

	Cursor and scroll position are synchronized between source and
	render windows: moving the cursor or scrolling on either side
	mirrors the other to the corresponding mapped line, with the
	cursor placed at the same window row so the views stay aligned.

	Closing the split window with `:q` or `<C-w>c` does NOT destroy
	the render buffer; it stays cached on the source for future
	toggles or splits.

	Limitation: inline images can only be bound to one window at a
	time.  Calling |:MdRender-split| twice yields two render windows,
	but only the most recent one displays inline images (the older
	render window still receives text-content live updates).

							     *:MdRender-pager*
:MdRender pager ~
	Pager mode.  Renders the current Markdown buffer in full-screen with
	no chrome.  Press `q` to quit Neovim.  Useful for browsing Markdown
	files like `less`: >bash

	  nvim +"MdRender pager" README.md
<
	Add a shell alias for convenience: >bash

	  alias mdless='nvim +"MdRender pager"'
	  mdless README.md
<

							      *:MdRender-demo*
:MdRender demo ~
	Show a demo window with all supported Markdown notations.

Deprecated commands				 *md-render-deprecated-commands*

The following top-level commands are kept as aliases for backwards
compatibility.  Each one prints a one-shot `vim.notify_once` warning the
first time it is invoked in a Neovim session, and forwards to the new
|:MdRender| dispatcher.  They will be removed in a future major version.

								  *:MdRenderTab*
:MdRenderTab ~
	Deprecated alias for `:MdRender tab`.

							     *:MdRenderToggle*
:MdRenderToggle ~
	Deprecated alias for `:MdRender toggle`.

							       *:MdRenderAuto*
:MdRenderAuto [on|off|toggle] ~
	Deprecated alias for `:MdRender auto [on|off|toggle]`.

							      *:MdRenderSplit*
:MdRenderSplit ~
	Deprecated alias for `:MdRender split`.  All Vim split modifiers
	(`:vert`, `:tab`, `:topleft`, `:botright`) work the same way.

							      *:MdRenderPager*
:MdRenderPager ~
	Deprecated alias for `:MdRender pager`.

							       *:MdRenderDemo*
:MdRenderDemo ~
	Deprecated alias for `:MdRender demo`.

TELESCOPE INTEGRATION					   *md-render-telescope*

							 *telescope.previewer()*
PREVIEWER ~

telescope.previewer({opts}) ~
	Create a previewer that can be passed to any telescope.nvim picker --
	builtin, extension, or custom.

	Parameters: ~
	  {opts}  (table|nil) Optional settings:
	    - {namespace}  (string) Extmark namespace name
				    (default: "md_render_telescope")

	Returns: ~
	  A telescope-compatible buffer previewer

	Example: >lua
	  local previewer = require("md-render.telescope").previewer()
	  require("telescope.builtin").find_files({ previewer = previewer })
	  require("telescope").extensions.egrepify.egrepify({
	    previewer = previewer,
	  })
<

The previewer handles three kinds of files:

  Markdown (.md, .markdown)     md-render rendering with highlights, links,
				and inline images.

  Image / Video                 Inline display via Kitty graphics protocol.
  (PNG, JPEG, WebP, GIF,        File name and dimensions are shown as a
  MP4, WebM, AVI, ...)          header.

  Other files                   Falls back to telescope's default previewer
				with syntax highlighting.

For grep-based pickers (e.g. `live_grep`, `grep_string`), the preview
automatically scrolls to the matched source line.

							  *:Telescope_md_render*
:TELESCOPE MD_RENDER EXTENSION ~

A shortcut for builtin pickers.  Wraps `telescope.builtin` pickers with the
md-render previewer.  All arguments are passed through.
>vim
  :Telescope md_render find_files
  :Telescope md_render live_grep cwd=~/notes
  :Telescope md_render grep_string search=TODO
<

	  })

  (PNG, JPEG, WebP, GIF,        File name and dimensions are shown as a

  Other files                   Falls back to snacks' default previewer with
				syntax highlighting.

For grep-based pickers the preview automatically scrolls to the matched

EXAMPLE							     *md-render-example*

A complete example showing how to render Markdown and display it in a buffer:
>lua

  local md = require("md-render")

  -- Render a single line of markdown
  local text, highlights, links =
    md.Markdown.render("**bold** and [link](https://example.com)")

  -- Build full document content
  local ContentBuilder = md.ContentBuilder
  local b = ContentBuilder.new()
  b:render_document(lines, {
    max_width = 80,
    indent = "  ",
    repo_base_url = "https://github.com/user/repo",
    autolinks = {
      {
        key_prefix = "JIRA-",
        url_template =
          "https://jira.example.com/browse/JIRA-<num>",
      },
    },
  })
  local content = b:result()

  -- Apply to a buffer
  local buf = vim.api.nvim_create_buf(false, true)
  local ns = vim.api.nvim_create_namespace("my_ns")
  md.display_utils.apply_content_to_buffer(buf, ns, content)

  -- Display images (Kitty Graphics Protocol terminal required)
  -- Images are automatically cleaned up when the window is closed.
  local win = vim.api.nvim_get_current_win()
  md.display_utils.setup_images(win, content, ns)

  (mmdc)               `npx -y @mermaid-js/mermaid-cli`.
