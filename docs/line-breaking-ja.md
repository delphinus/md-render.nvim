# 日本語テキストの行折り返し処理

md-render.nvim では、日本語テキストを自然に折り返すために3段階のセグメント分割を行っています。

## 処理の流れ

```
テキスト
  │
  ▼
┌─────────────────────────────────┐
│ 1. BudouX によるフレーズ分割    │  「携帯電話の利用についてのお知らせ」
│    (budoux.lua + budoux_ja.lua) │  → 「携帯電話の」「利用に」「ついての」「お知らせ」
└─────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────┐
│ 2. 文字種境界による細分割       │  「携帯電話の」→「携帯電話」+「の」
│    (budoux.lua split_by_script) │  「参照してください」→「参照」+「してください」
└─────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────┐
│ 3. JIS X 4051 禁則処理          │  行頭禁則：「。」「、」「）」等が行頭に来ない
│    (content_builder.lua)        │  行末禁則：「（」「「」等が行末に来ない
└─────────────────────────────────┘
  │
  ▼
折り返し済みテキスト
```

## 1. BudouX によるフレーズ分割

[BudouX](https://github.com/google/budoux)（Google 製、Apache-2.0 ライセンス）の日本語モデルを Lua に移植して使用しています。機械学習モデルにより、テキスト中の自然な分割位置を判定します。

### アルゴリズム

各文字間の位置について、周囲6文字の特徴量（ユニグラム・バイグラム・トライグラム）からスコアを計算し、スコアが正なら分割します。

```
文字列: 今 日 は と て も 良 い 天 気 で す ね 。
                ↑        ↑     ↑
              分割     分割  分割
結果:  「今日は」「とても」「良い」「天気ですね。」
```

### ファイル構成

- `lua/md-render/budoux.lua` — パーサー本体（`parse` 関数）と文字種分割（`split_by_script` 関数）
- `lua/md-render/budoux_ja.lua` — 日本語モデルデータ（[budoux/models/ja.json](https://github.com/google/budoux/blob/main/budoux/models/ja.json) から生成）

### 効果

BudouX を使わない場合、CJK 文字は1文字ずつ独立したセグメントになります。

```
BudouX なし: 東|京|都|の|天|気|は|晴|れ|で|す  （1文字ずつ）
BudouX あり: 東京都の|天気は|晴れです           （意味のある単位）
```

1文字ずつの分割でも行幅は守れますが、BudouX を使うと単語の途中で改行しない、より読みやすい折り返しになります。

## 2. 文字種境界による細分割

BudouX のフレーズは長い場合があります（例:「参照してください」= 18桁）。行末に収まらないとき、フレーズ全体が次の行に送られてしまい、前の行に不自然な余白が生じます。

これを防ぐため、BudouX のフレーズをさらに文字種の境界で細分割します。

### 分割ルール

| 遷移 | 分割 | 例 |
|---|---|---|
| 漢字2文字以上 → ひらがな | する | `参照` + `してください` |
| 漢字1文字 → ひらがな | しない | `起こる`（送り仮名） |
| 漢字 → カタカナ | する | `翻訳` + `データ` |
| カタカナ → ひらがな | する | `プラグイン` + `です` |
| カタカナ → 漢字 | する | `データ` + `管理` |
| ひらがな → 漢字 | しない | `お知らせ` |
| ひらがな → カタカナ | しない | `高いマスタデータ` |
| 中黒「・」の後 | する | `ユニグラム・` + `バイグラム` |

### 中黒「・」による分割

中黒「・」（U+30FB）はカタカナ複合語の区切りとして使われます（例:「フロント・バックエンド」）。「・」の直後を分割ポイントとし、「・」自体は前のチャンクの末尾に残します。

```
分割前: ユニグラム・バイグラム・トライグラム  （1つの塊）
分割後: ユニグラム・|バイグラム・|トライグラム  （「・」の後で分割）
```

「・」は JIS X 4051 の行頭禁則文字（NO_BREAK_START）にも含まれているため、行頭に来ることはありません。

### 漢字1文字の特別扱い

漢字→ひらがなの遷移で分割するのは、漢字が2文字以上続いている場合のみです。

- **漢字2文字以上 + ひらがな** — 漢語の語幹 + 助詞・助動詞の組み合わせ（`参照/する`、`管理/して`）
- **漢字1文字 + ひらがな** — 和語の送り仮名（`起こる`、`食べる`、`渡さない`、`美しい`）

漢語は語幹と活用部分が独立しているため分割しても読みやすさを損ないませんが、和語は漢字と送り仮名が一体で意味をなすため、分割すると読みにくくなります。

### 句読点・記号の扱い

句読点や記号（`。`、`、`、`（`など）は「O」（その他）クラスとして扱い、分割のトリガーにしません。直前のセグメントに付随します。

### wrap_words との関係

細分割されたセグメントは `has_leading_space = false`（スペースなし）で隣接するため、`wrap_words` は行に余裕がある限り連結します。行幅を超える場合にのみセグメント境界で改行します。

```
行幅に余裕あり: 「参照」+「してください」→ 「参照してください」（連結）
行幅が不足:     「...を参照」 / 「してください。」（境界で改行）
```

## 3. JIS X 4051 禁則処理

最終段階として、`wrap_words`（`content_builder.lua`）が JIS X 4051 に基づく禁則処理を適用します。

### 行頭禁則文字（NO_BREAK_START）

行の先頭に置いてはいけない文字です。

- 終わり括弧: `）` `〕` `］` `｝` `〉` `》` `」` `』` `】` `»`
- 句読点: `。` `．` `、` `，`
- 区切り約物: `！` `？` `‼` `⁇` `⁈` `⁉`
- 中点: `・` `：` `；`
- 長音: `ー`
- 小書き仮名: `ぁ` `ぃ` `ぅ` `っ` `ゃ` `ゅ` `ょ` 等
- 繰返し記号: `々` `ゝ` `ヽ` 等

### 行末禁則文字（NO_BREAK_END）

行の末尾に置いてはいけない文字です。

- 始め括弧: `（` `〔` `［` `｛` `〈` `《` `「` `『` `【` `«`

### 処理方式

- **追い出し（push-out）**: 行頭禁則文字が次の行の先頭に来る場合、前の行の末尾セグメントごと次の行に送る
- **追い込み（push-in）**: 追い出しができない場合（前の行が空になる場合）、禁則文字を前の行に残す（行幅を超えることを許容）

## BudouX と禁則処理の関係

BudouX と JIS 禁則処理は競合せず、補完的に機能します。

| 処理 | 役割 | レベル |
|---|---|---|
| BudouX | テキストを意味のある単位に分割 | フレーズ |
| 文字種分割 | 長いフレーズを語幹・活用で細分割 | 単語 |
| JIS 禁則 | 特定文字の行頭・行末配置を制御 | 文字 |

BudouX のモデルは暗黙的に句読点の扱いを学習していますが、JIS 禁則ルールを最終段階で適用することで、モデルがカバーしきれないケースにも対応できます。
